Title: Neural Network Architectures and Activations

  • The Big Leap: Moving from linear regression to hierarchical, non-linear models.
  • Unit Role: Understanding the “Atoms” (neurons) and “Soul” (activations) of Deep Learning.
  • Goal: Build a solid intuition for why and how multi-layer networks work.

Learning outcomes

By the end of this unit, students can:

  • Explain the transition from biological neurons to the McCulloch-Pitts and Rosenblatt Perceptron models.
  • Prove why a multi-layer linear network is mathematically equivalent to a single linear layer.
  • Compare modern activation functions (ReLU, GeLU, etc.) and their impact on gradient flow.
  • Formulate the Universal Approximation Theorem and its implications for materials modeling.

Recap from Unit 3

  • Loss Minimization: We’ve seen how to fit parameters by minimizing a cost function.
  • Basis Functions: We used fixed \(\phi(x)\). Now, we will let the network learn its own basis functions!
  • Non-linearity: Previously external; now embedded in every unit of the model.

Why Neural Networks?

  • Manual Feature Engineering is hard: In complex materials data (microstructures, multi-modal spectra), we don’t know the best features.
  • Hierarchical Representations: Networks learn simple features in early layers and combine them into complex concepts in later layers [@ryan2021machine].
  • Scalability: They scale gracefully with both the number of samples and the dimensionality of features.

The Biological Inspiration

  • Biological Neuron: Dendrites (inputs), Cell Body (summation), Axon (output), Synapses (weights).
  • History: The brain as a logical machine.
  • Caveat: Modern Artificial Neural Networks (ANNs) are inspired by biology but are primarily mathematical optimization tools [@sandfeld_materials_data_science].

The McCulloch-Pitts Model (1943)

  • The First Model: Binary inputs, binary threshold output.
  • Logic Gates: Showed that networks of these units could perform AND, OR, and NOT operations.
  • Limitation: Weights were fixed; it couldn’t “learn” from data.

Rosenblatt’s Perceptron (1958)

  • The Innovation: Added a learning rule to adjust weights based on errors.
  • Architecture: Single layer mapping inputs directly to a binary output.
  • Perceptron Convergence Theorem: If the data is Linearly Separable, the algorithm will find a solution in finite steps.

The XOR Crisis

graph LR
    X1((x1)) --> H1((h1))
    X1((x1)) --> H2((h2))
    X2((x2)) --> H1
    X2((x2)) --> H2
    H1 --> Y((y))
    H2 --> Y
  • Minsky & Papert (1969): Proved that a single-layer perceptron cannot solve the XOR problem (non-linearly separable).
  • The Solution: Hidden layers and non-linear activations.
  • Impact: Led to the first “AI Winter” but paved the way for the Multi-Layer Perceptron (MLP).

The ADALINE Model

  • Adaptive Linear Neuron: Used a linear activation for training (LMS rule) and a threshold for decision.
  • Key Contribution: Separated the error signal from the final discrete decision, allowing for smooth gradient-based optimization [@neuer2024machine].

The Artificial Neuron (Node)

graph LR
    X1((x1)) -- W1 --> Sum[Σ]
    X2((x2)) -- W2 --> Sum
    dots[...] --> Sum
    XD((xD)) -- WD --> Sum
    B((1)) -- b --> Sum
    Sum --> Act[σ]
    Act --> Out((a))
  • The Formula: \(z = \mathbf{W}\mathbf{x} + \mathbf{b}\) (Affine transformation).
  • Activation: \(a = \sigma(z)\) (Non-linear mapping).
  • Parameters: \(\mathbf{W}\) (weights) determine the slope/importance; \(\mathbf{b}\) (bias) determines the threshold/shift.

Role of the Bias Term

  • Without a bias, the hyperplane must pass through the origin.
  • Geometric View: The bias provides an extra degree of freedom to shift the decision boundary in the input space.
  • Vectorized Trick: Append a constant ‘1’ to the input vector \(\mathbf{x}\) and include \(\mathbf{b}\) in the weight matrix \(\mathbf{W}\).

Non-linearity: The Soul of the Network

  • Linearity Theorem: A composition of linear functions is still linear: \(f(g(\mathbf{x})) = \mathbf{A}(\mathbf{B}\mathbf{x}) = (\mathbf{A}\mathbf{B})\mathbf{x}\).
  • Collapse: Without non-linear activations, a 1000-layer network is just a single linear regression model.
  • Functionality: Activations allow the network to “bend” space and fit non-convex manifolds.

Activation Function Taxonomy

Classic (Bounded)

  • Sigmoid: Range (0, 1), probabilistically interpretable.
  • Tanh: Range (-1, 1), zero-centered.
  • Softmax: Multi-class probability (output only).

Modern (Unbounded)

  • ReLU: Fast, sparse, “dying” issue.
  • Leaky ReLU: Small slope for negative values.
  • GeLU: Gaussian-weighted (Transformers).

// Define the functions and derivatives
actFns = {
  return {
    "Sigmoid": {f: z => 1 / (1 + Math.exp(-z)), df: z => {let s = 1/(1+Math.exp(-z)); return s*(1-s);}},
    "Tanh": {f: z => Math.tanh(z), df: z => 1 - Math.pow(Math.tanh(z), 2)},
    "ReLU": {f: z => Math.max(0, z), df: z => z >= 0 ? 1 : 0},
    "Leaky ReLU": {f: z => z > 0 ? z : 0.1*z, df: z => z >= 0 ? 1 : 0.1},
    "ELU": {f: z => z > 0 ? z : 1.0*(Math.exp(z)-1), df: z => z >= 0 ? 1 : 1.0*Math.exp(z)},
    "GeLU": {
      f: z => z * 0.5 * (1 + Math.tanh(Math.sqrt(2 / Math.PI) * (z + 0.044715 * Math.pow(z, 3)))),
      // approximate derivative
      df: z => {
        let h = Math.sqrt(2/Math.PI) * (z + 0.044715 * Math.pow(z, 3));
        let cdf = 0.5 * (1 + Math.tanh(h));
        let pdf = Math.exp(-0.5 * z * z) / Math.sqrt(2 * Math.PI);
        return cdf + z * pdf;
      }
    }
  };
}

Logistic Sigmoid

  • \(\sigma(z) = \frac{1}{1+e^{-z}}\)
  • Range: (0, 1). Interpretable as a probability.
  • Downside: Vanishing Gradients. For large \(|z|\), the derivative \(\sigma'(z) \approx 0\), stopping the learning process.

Hyperbolic Tangent (Tanh)

  • \(\tanh(z) = \frac{e^z - e^{-z}}{e^z + e^{-z}}\)
  • Range: (-1, 1). Zero-centered, which often leads to faster convergence than sigmoid.
  • Downside: Still suffers from vanishing gradients in deep networks.

Rectified Linear Unit (ReLU)

  • \(\text{ReLU}(z) = \max(0, z)\)
  • The Game Changer: No saturation for positive values.
  • Efficiency: Very fast to compute (just a comparison).
  • Sparse Activations: Naturally leads to many neurons being “off,” which can act as a regularizer.

The “Dying ReLU” Problem

  • If a neuron’s weights are adjusted such that it always outputs 0, the gradient becomes 0, and it can never recover.
  • Solution: Careful initialization or variants like Leaky ReLU.

Leaky ReLU and PReLU

  • \(\text{Leaky ReLU}(z) = \max(\alpha z, z)\)
  • Parametric ReLU (PReLU): \(\alpha\) is a learned parameter.
  • Effect: Keeps the gradient flow alive for negative values, making training more robust.

Exponential Linear Unit (ELU)

  • \(\text{ELU}(z) = \begin{cases} z & z > 0 \\ \alpha(e^z - 1) & z \leq 0 \end{cases}\)
  • Benefit: Smoothly approaches -\(\alpha\) for negative values, providing more noise-robustness and faster convergence than standard ReLU.

GeLU (Gaussian Error Linear Unit)

  • \(\text{GeLU}(x) = x \Phi(x)\)
  • Modern Standard: Used in GPT and BERT.
  • Intuition: Weighting the input by its probability of being dropped out under a Gaussian noise assumption.

Softmax: The Classification Engine

  • \(\text{Softmax}(\mathbf{z})_j = \frac{e^{z_j}}{\sum_k e^{z_k}}\)
  • Properties: Outputs sum to 1 and are all positive.
  • Interpretation: Multi-class probability distribution.
  • Position: Almost always the final layer in a classification network.

Layer-by-Layer Architecture

  • Input Layer: Pass-through layer for raw features.
  • Hidden Layers: Layers where the “magic” (representation learning) happens.
  • Output Layer: Final prediction (Continuous for regression, Softmax for classification).

Depth vs. Width

  • Width: Number of neurons in a layer. Affects memory and the number of features learned at one level.
  • Depth: Number of hidden layers. Affects the level of abstraction and complexity.
  • Deep Trend: Deep, narrow networks often generalize better and require fewer total parameters than shallow, wide ones.

Multi-Layer Perceptron (MLP)

  • Also called Fully Connected (FC) or Dense networks.
  • Every neuron in layer \(l\) is connected to every neuron in layer \(l-1\).
  • Vectorized Notation: \(\mathbf{a}^{(l)} = \sigma(\mathbf{W}^{(l)}\mathbf{a}^{(l-1)} + \mathbf{b}^{(l)})\).

Forward Propagation Summary

  1. Input: \(\mathbf{a}^{(0)} = \mathbf{x}\)
  1. For \(l = 1 \dots L\): \(\mathbf{z}^{(l)} = \mathbf{W}^{(l)}\mathbf{a}^{(l-1)} + \mathbf{b}^{(l)}\) \(\mathbf{a}^{(l)} = \sigma(\mathbf{z}^{(l)})\)
  1. Output: \(\hat{\mathbf{y}} = \mathbf{a}^{(L)}\)

Matrix Dimensions Checklist

  • If input \(\mathbf{x}\) is \(D \times 1\) and layer \(l\) has \(M\) neurons:
  • \(\mathbf{W}^{(l)}\) must be \(M \times D\).
  • \(\mathbf{b}^{(l)}\) must be \(M \times 1\).
  • \(\mathbf{a}^{(l)}\) will be \(M \times 1\).
  • Batching: If we pass \(N\) samples, \(\mathbf{X}\) is \(D \times N\), and outputs are \(M \times N\).

Universal Approximation Theorem (Cybenko, 1989)

  • The Statement: A feed-forward network with a single hidden layer and a non-linear activation can approximate any continuous function on a compact subset of \(\mathbb{R}^D\) to arbitrary precision.
  • Catch: It doesn’t say how many neurons you need (it could be millions!) or how to find the weights.

Expressivity vs. Learnability

  • Expressivity: Can the architecture represent the solution? (Yes, per UAT).
  • Learnability: Can our optimization algorithm (SGD) find that solution in a reasonable time? (Much harder!).

Weight-Space Symmetries

  • In an MLP, you can swap two hidden neurons in the same layer (and their corresponding incoming/outgoing weights) without changing the function.
  • Implication: The loss landscape is highly redundant with many equivalent global minima.

Initialization: The Symmetry Problem

  • If all weights are initialized to the same value (e.g., 0), all neurons in a layer will learn the exact same feature.
  • Symmetry Breaking: We must initialize weights randomly to ensure neurons specialize in different patterns.

Xavier/Glorot Initialization

  • Goal: Keep the variance of activations and gradients constant across layers.
  • Rule: Sample weights from \(\mathcal{N}(0, \frac{2}{n_{in} + n_{out}})\).
  • Standard: The default for Tanh and Sigmoid activations.

He Initialization

  • Optimized for ReLU: Because ReLU “kills” half the variance (the negative part), we need a larger initial variance.
  • Rule: Sample weights from \(\mathcal{N}(0, \frac{2}{n_{in}})\).
  • Crucial: Prevents vanishing gradients in very deep ReLU networks.

Feature Learning Intuition

  • Layer 1: Detects edges or simple gradients in the data.
  • Layer 2: Detects shapes or motifs (e.g., grain boundaries, phase clusters).
  • Layer 3: Detects high-level concepts (e.g., “cracked microstructure”).
  • Composition: Complexity through hierarchy.

Visualizing Activations

  • By plotting the activations \(\mathbf{a}^{(l)}\) as images, we can “see” what the network is looking at.
  • Materials Science: Often shows the network focusing on dislocations, lattice defects, or chemical interfaces [@sandfeld_materials_data_science].

Over-parameterization

  • Modern networks often have many more parameters than data points.
  • Surprisingly, they still generalize! This is the “Double Descent” mystery we will explore in Unit 7.

Computational Graphs

  • A network is a directed acyclic graph (DAG) of operations.
  • This abstraction allows modern frameworks (TensorFlow, PyTorch) to automate the differentiation process (Autograd).

Hardware Acceleration

  • Matrix-matrix multiplication (\(\mathbf{W} \times \mathbf{X}\)) is the most compute-intensive part.
  • GPUs: Thousands of cores performing these multiplications in parallel.
  • TPUs: Specialized hardware for tensor operations.

Common failure mode #1: Lack of Normalization

  • If inputs are in different ranges (e.g., Temperature in [0, 1000] and Pressure in [0, 1]), the loss landscape will be highly distorted.
  • Result: Stalled learning or extreme sensitivity to learning rate.

Common failure mode #2: Wrong Activation for Output

  • Using ReLU for an output that can be negative.
  • Using Sigmoid for an output that isn’t a probability.
  • Result: Biased or physically impossible predictions.

Common failure mode #3: Saturated Sigmoids

  • Initializing weights too large causes Sigmoids to start in the “flat” region.
  • Result: Gradient is zero from slide 1; nothing ever learns.
