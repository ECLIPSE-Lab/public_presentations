01. Title: Latent Spaces and Embeddings

  • The Hidden Dimension: Most physical properties are not directly observed but are emergent from a hidden, “latent” structure.
  • Unit Role: Mastering the geometry and statistics of the hidden representations we learned in Unit 9.
  • Goal: Transition from “finding” representations to “understanding and navigating” them.

02. Learning outcomes

By the end of this unit, students can: - Define Latent Variables and explain their role in modeling complex systems. - Compare t-SNE and UMAP for high-dimensional visualization and structural analysis. - Formulate the Embedding process for mapping discrete materials data to continuous spaces. - Use latent distributions to detect Anomalies and quantify Novelty in process data.

03. Recap from Unit 9

  • Representation Learning: We trained Autoencoders to find a low-dim code \(\mathbf{h}\).
  • Manifold: We assumed the data lives on a curved surface.
  • Missing: We haven’t looked closely at the “neighborhood” structure of this code.

04. What is a Latent Variable?

  • A variable that is not directly measured but is inferred from other variables that are measured.
  • Physics Analogy: Temperature is a latent summary of the kinetic energy of millions of particles.
  • ML Analogy: “Quality” is a latent summary of 50 different sensor readings.

05. Discrete vs. Continuous Latent Variables

  • Discrete: Cluster membership, phase ID, defect type.
  • Continuous: Intrinsic coordinates on a manifold, concentration gradients, stress states.
  • This unit focuses on the Continuous case and its geometric properties.

06. The Geometry of Similarity

  • In a “good” latent space, Euclidean distance \(\approx\) Semantic similarity.
  • Two micrographs of the same phase should be “close.”
  • A micrograph of a brittle fracture and one of a ductile fracture should be “far.”

07. The Mapping Problem

  • How do we project 1000-D data to 2-D while preserving the “clusters”?
  • Linear (PCA): Preserves large distances (global structure) but squashes local clusters.
  • Non-linear (Manifold Learning): Focuses on preserving local relationships.

08. t-SNE: The Visualization King

  • t-Distributed Stochastic Neighbor Embedding.
  • The Core Idea: Convert distances in high-dim space into probabilities of being neighbors. Do the same in low-dim space. Minimize the difference.
  • Similarity: \(p_{j|i} = \frac{\exp(-\|x_i - x_j\|^2 / 2\sigma_i^2)}{\sum \dots}\).

09. The Crowding Problem

  • In high dimensions, there is “more room” for neighbors.
  • When we squash data to 2D, the neighbors want to be in the same spot, leading to a “crushed” visualization.
  • The t-SNE Solution: Use a heavy-tailed Student’s t-distribution in the low-dim space.

10. t-SNE Objective: KL-Divergence

  • \(J = \sum_i D_{KL}(P_i || Q_i) = \sum_i \sum_j p_{j|i} \log \frac{p_{j|i}}{q_{j|i}}\).
  • Behavior: It penalizes putting high-dim neighbors far apart in 2D, but cares less about putting far-away points close together.
  • Result: Excellent at identifying “islands” or clusters (Murphy 2012).

11. Hyperparameter: Perplexity

  • Perplexity \(\approx\) The number of nearest neighbors each point considers.
  • Too Low: Data looks like a random cloud of tiny dots.
  • Too High: All clusters merge into one giant blob.
  • Golden Rule: Always try multiple perplexity values (e.g., 5, 30, 100).

12. Limits of t-SNE

  • Global Structure: t-SNE is famous for breaking global relationships (the distance between “Islands” is often meaningless).
  • Computation: \(O(N^2)\), making it slow for very large datasets.
  • Non-parametric: You cannot easily “project” a new point into an existing t-SNE map.

13. UMAP: The Modern Alternative

  • Uniform Manifold Approximation and Projection.
  • Based on Topological Data Analysis (simplicial complexes).
  • Benefit: Much faster than t-SNE and preserves global structure significantly better.

14. UMAP Geometry

  • Assumes the data is uniformly distributed on a Riemannian manifold.
  • It builds a graph of the data and finds a low-dim layout that has the same fuzzy topological structure.
  • Result: You can trust both the local clusters AND the large-scale distances between them (Sandfeld et al. 2024).

15. What is an Embedding?

  • A specific type of latent representation where discrete or sparse data is mapped to a dense vector space.
  • Input: “Fe”, “Cu”, “Al” (categories).
  • Output: \([0.1, -0.5, \dots]\) (embeddings).
  • Learning: The network learns that “Fe” and “Ni” vectors should be close because they have similar properties.

16. Word2Vec Intuition for Materials

  • “You shall know a word by the company it keeps.”
  • Materials Context: You shall know an element by the crystal environments it occupies.
  • Embedding: Elements that can substitute for each other in a lattice will have similar latent vectors.

17. Entity Embeddings

  • Mapping categorical variables (e.g., Supplier ID, Machine Type) to continuous space.
  • Discovery: The network might discover that Machine A and Machine C are “similar” (close in embedding space) because they both produce similar defect patterns.

18. Latent Space Distribution (\(p(\mathbf{h})\))

  • After training an AE, we can plot the histogram of each latent neuron.
  • Is it Gaussian? Often, it’s highly skewed or multi-modal.
  • Meaning: Gaps in the latent distribution represent “physical impossibilities” or “unexplored material regimes.”

19. Monitoring the Latent State

  • In an industrial process, we map multi-sensor data to a latent code \(\mathbf{h}_t\).
  • Nominal State: A high-density region in latent space.
  • Degradation: The code slowly drifts toward the edge of the distribution.
  • Failure: The code jumps to an empty region (Neuer et al. 2024).

20. Conditional Latent Probability

  • \(P(h_2 | h_1)\): If the first latent feature (e.g., grain size) is \(X\), what is the likely value of the second feature (e.g., hardness)?
  • This allows for Latent Reasoning without explicitly using the high-dim pixels or spectra.

21. Latent Interventions

  • “Nudging” a point in latent space and decoding it to see the physical result.
  • Example: Moving along the “corrosion” axis in latent space to predict what a sample will look like in 5 years.

22. Metric Learning in Latent Space

  • We can force the latent space to respect a physical metric (e.g., energy difference).
  • Result: The “distance” in the embedding space is exactly equal to the physical work required to transform material A into material B.

23. Disentanglement Analysis

  • Checking if latent neurons are correlated.
  • Ideal: Diagonal covariance matrix in latent space. Each neuron controls one independent physical lever.

24. Scaling to Millions of Points

  • Techniques for approximate nearest neighbors (Annoy, Faiss) are needed to handle massive latent libraries.
  • Materials Database: Searching for the “closest neighbors” to a new alloy in a latent space of 100 million compounds.

25. Evaluating Latent Quality

  • Reconstruction Accuracy: Does \(\mathbf{h}\) keep the data?
  • Clustering Quality: Does \(\mathbf{h}\) separate known phases?
  • Downstream Performance: Can we predict properties using only \(\mathbf{h}\) as input?

26. Feature Correlation in Latent Space

  • Even if input features are correlated, a good representation learner (Unit 9) should provide orthogonal latent features.
  • This effectively “un-mixes” the physics of the problem.

27. Reliability of Visualizations

  • Caution: Just because points are clustered in t-SNE doesn’t mean they are a physical cluster.
  • Always verify clusters using density-based methods (DBSCAN) in the full-dim latent space, not just the 2D map.

28. Manifold Curvature and Distortions

  • Every projection from high-dim to low-dim introduces distortion.
  • Check: Compare the pairwise distance matrix in high-dim vs. low-dim. If they correlate well, the embedding is “faithful.”

29. Latent Space for Active Learning

  • Pick new experiment points where the latent density is lowest (Uncertainty).
  • This ensures we explore the entire physical manifold with minimum samples (Neuer et al. 2024).

30. From Embeddings to Design

  • Optimization doesn’t happen in “atom space” (too discrete/complex).
  • It happens in the continuous Embedding Space.
  • Find \(\mathbf{h}^*\) that maximizes strength, then decode to get the recipe.

31. The Semantic Manifold of Physics

  • Data points are not just numbers; they are “sentences” in the language of physical laws.
  • Latent spaces are the grammar of this language.

32. Analyzing Cluster Overlap

  • If “High Strength” and “Low Cost” clusters overlap in latent space, a Pareto-optimal solution exists in that region.

33. Latent Signatures of Failure

  • Certain trajectories in latent space are “death spirals.”
  • Predictive Maintenance: Detecting the signature of a bearing failure 100 hours before it happens.

34. Domain-Informed Embeddings

  • Initializing the embedding layer with known property vectors (e.g., electronegativity, atomic radius) rather than random noise.
  • Effect: Speeds up training and ensures physical consistency.

35. Hierarchical Latent Spaces

  • Mapping samples \(\to\) Morphology \(\to\) Physics.
  • Using nested Autoencoders to represent multiscale phenomena (e.g., from atoms to grains to parts).

36. Materials example: Alloy Embeddings

  • Input: One-hot encoded composition.
  • Latent: 16-D vector.
  • Result: The latent space recreates the Mendeleev Table logic purely from alloy property data (McClarren 2021).

37. Materials example: SEM Texture Map

  • Using UMAP to map \(10,000\) small image patches.
  • Discovery: Identifying “Transition Zones” between different phases that are invisible to the naked eye.

38. Materials example: Anomaly in Plasma Process

  • Mapping voltage/current spectra to 2D.
  • Observation: During a stable run, the code stays in a “circle.” When a contamination occurs, it shoots out into a “tail” (Neuer et al. 2024).

39. Common failure mode #1: Misleading Clusters

  • t-SNE “finding” clusters in perfectly random Gaussian noise.
  • Diagnostic: Run t-SNE multiple times with different seeds. If the clusters change position wildly, they are hallucinations.

40. Common failure mode #2: Over-Interpretation of 2D

  • Assuming that because A is “closer” to B than C in 2D, the same is true in the full latent space.
  • Check: Always verify with the original high-dim distance matrix.

41. Common failure mode #3: Perplexity Mismatch

  • Using a perplexity of 5 for a dataset of 1 million points.
  • Result: A noisy “hairball” with no structure.

47. Exam-oriented key statements

  1. Latent variables summarize hidden structural causes.
  2. t-SNE uses the t-distribution to solve the crowding problem.
  3. UMAP preserves both local and global structure via topology.
  4. Embeddings map discrete categories to dense, searchable continuous spaces.
  5. Latent density monitoring is a powerful tool for process safety.
