# Unit 4 Plan — Databases, Ontologies, and FAIR (MG)

## Unit intent
Build the data infrastructure perspective required to make MG pipelines reproducible, interoperable, and trustworthy.

## Learning objectives
1. Explain core database/provenance requirements for MG datasets.
2. Understand ontology benefits for cross-lab semantic consistency.
3. Apply FAIR principles to evaluate and improve dataset quality.
4. Design leakage-resistant ingestion and split workflows using metadata.

## Dependency links
- Backward: Unit 2 (structure data), Unit 3 (representations/descriptors)
- Forward: Unit 5 (supervised learning requires robust data plumbing)

## Slide architecture (50)
- 1–14: Data lifecycle, sources, metadata, provenance
- 15–24: Ontology and interoperability basics
- 25–36: FAIR pillars + governance + validation
- 37–42: Case sketches and dependency mapping
- 43–50: Exercise scaffolds + summary + references

## Exercise scaffold
- E1: Build a metadata inventory for one dataset
- E2: Define schema + validation checks
- E3: FAIR assessment + leakage audit + split proposal
